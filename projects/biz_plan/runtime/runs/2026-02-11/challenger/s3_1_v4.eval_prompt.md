3-1. 비즈니스 모델 및 수익 구조

AI기반 마이디지털정보 오남용 알림 서비스는 사용자가 인지하지 못한 개인정보와 사진·동영상의 노출을 탐지하고, 케이스 단위로 증거 확보와 조치 착수를 연결하는 구독형 보호 서비스로 설계한다.[R1] 개인정보 유출 신고 동향이 공표되는 환경에서 개인의 대응은 채널 분산과 수동 절차로 인해 지연되기 쉬우며, 이 지연이 고객 손실로 전이된다.[R2] 본 섹션의 수익 구조는 탐지와 조치가 케이스 단위로 운영된다는 전제를 과금 단위와 원가 단위에 정합시키고, As-of 기준에서 확인 가능한 사업화 상태와 확인 불가 항목을 동일 표준으로 제시하는 데 목적이 있다.[R1]

3-1-1. 구독 기반 수익모델(요금제·가치교환·과금 단위·업셀 경로)

앞서 정의한 케이스 단위 운영을 매출로 연결하기 위해, 개인 구독 요금제는 과금 단위를 계정 월 과금으로 고정하고 케이스 연동 서비스는 별도 과금 경로로 분리한다.[R1] 요금제의 가치교환은 탐지 범위와 조치 지원 깊이의 단계화로 구성하며, 개인정보 보호 관련 의무와 대응 증적의 요구사항을 반영해 알림과 조치 산출물을 함께 제공하는 것을 전제로 한다.[R3][R1] 투자자 관점의 의사결정 포인트는 월 구독료가 사용자 수 연동 변동비를 커버하고, 케이스 연동 변동비는 별도 과금 또는 상위 요금제 업셀로 회수되는 구조로 설계되어 있는지 여부다.[R1]

요금제 상태 월 요금 제공 가치교환 요약 과금 단위 업셀/크로셀 경로 근거 [R#]
베이직 2024.10 계획 [R1] 4,900원/월 [R1] SNS 및 웹 사진·영상 유출 감지, 개인정보 유출 감지, 기본 경고 알림 [R1] 개인 구독 계정 월 과금 [R1] 프리미엄 업셀, VIP 업셀 [R1] [R1]
프리미엄 2024.10 계획 [R1] 9,900원/월 [R1] 딥웹 및 다크웹 유출 감지, 게시물 삭제 요청 대행, AI 안전 필터 [R1] 개인 구독 계정 월 과금, 삭제 요청 대행 초과분 건 과금 [R1] VIP 업셀, 삭제 요청 대행 초과분 건 과금 크로셀 [R1] [R1]

ARPU는 개인 구독 기준으로 정의하며, ARPU = 요금제별 월요금의 믹스 가중합으로 산출한다.[R1] 업셀 경로는 노출 탐지 범위 확대, 삭제 요청 대행 필요도, 긴급 대응 필요도에 의해 베이직에서 프리미엄과 VIP로 이동하도록 설계한다.[R1] As-of 기준에서 요금제 실제 적용 여부와 요금제 믹스는 결제 로그 및 구독 상태 로그가 추가로 제공되지 않아 확인 불가로 처리한다.[R1]

3-1-2. 추가 수익원(B2B 제휴·기업 계약·부가서비스 과금·정산 구조)

개인 구독 요금제가 사용자 수 기반 매출을 구성한다면, 다음 단계는 케이스 발생과 조직 단위 수요를 흡수하는 추가 수익원을 정의해 요금제 외 매출원을 확보하는 것이다.[R1] 통신사 부가서비스 제휴는 통신사 청구 매출을 기준으로 계약서 정의 정산 규칙에 따라 수익 배분하는 구조로 설계하며, 정산 데이터는 제휴 정산 리포트와 계약서로 고정한다.[R1] 기업 임직원 보호 서비스는 연간 계약 단위로 정산하고 ARPA는 기업 정산 매출을 유효 계약 수로 나눈 값으로 정의하며, 계약 범위와 과금 단위는 계약서로 확정되는 것을 전제로 한다.[R1]

부가서비스 과금은 프리미엄에 포함된 삭제 요청 대행 범위를 초과하는 케이스를 건 단위로 과금하여, 케이스 연동 변동비를 회수하는 구조로 설계한다.[R1] 삭제 요청 대행은 외부 플랫폼의 개인정보 삭제 요청 절차를 대행·추적하는 과정에서 인력 및 외부 대응 비용이 케이스 수에 비례해 발생하므로, 케이스 로그와 요청 결과 리포트를 매출 인식 및 원가 인식의 공통 기준으로 사용한다.[R4][R1] 투자자 관점의 의사결정 포인트는 제휴·기업 계약·건당 과금 각각이 어떤 변동비 동인을 상쇄하도록 설계되어 있는지와, 정산 근거가 되는 리포트와 계약서가 통제된 형태로 존재하는지 여부다.[R1]

3-1-3. 목표시장 진입 현황 및 사업화 추진성과(파일럿/테스터/출시 일정/다음 단계)

추가 수익원 설계가 실현되었는지 판단하려면, 계획과 관측을 As-of 기준의 확인 가능 상태로 전환할 수 있는지부터 점검해야 한다.[R1] 내부 문서에는 파일럿 테스터와 만족도, 출시 일정 계획이 기재되어 있으나, As-of 기준 실측을 단정할 수 있는 결제 로그, 릴리즈 로그, 정산 리포트가 추가로 제공되지 않아 확인 불가 항목이 존재한다.[R1] 따라서 본 절은 계획 대비 확인 가능 상태를 표로 고정하고, 다음 의사결정에 필요한 데이터 원천을 명시한다.[R1]

다음 단계의 사업화 검증은 릴리즈 로그로 출시 여부를, 결제 로그로 유료 전환과 매출을, 케이스 로그로 처리량과 건당 원가를 실측으로 전환하는 순서로 수행한다.[R1] 특히 삭제 요청 대행과 긴급 대응은 케이스 단위로 원가와 매출이 동시 발생하므로, 케이스 식별자 기반의 로그 결합이 우선 과제다.[R1] 투자자 관점의 의사결정 포인트는 표의 확인 불가 항목을 실측으로 전환할 때 필요한 데이터 접근 권한과 산출 규칙이 내부 통제 문서로 고정되어 있는지 여부다.[R1]

3-1-4. 단위경제 및 손익 구조(원가 구성·총마진·CAC/LTV/Payback·손익분기, 단위경제 요약표 포함)

사업화 상태에 확인 불가가 남아 있는 구간에서는, 단위경제는 수치 제시가 아니라 과금 단위와 원가 동인의 일치 여부를 먼저 고정해야 한다.[R1] 변동비는 사용자 수에 비례하는 탐지·저장·모니터링 비용과 케이스 수에 비례하는 삭제 요청 대행 처리 비용, 법률 상담 및 사고 대응 연계 비용으로 구분하며, 고정비는 모델·제품 개발 인력, 운영 체계, 공통 인프라 비용으로 구분한다.[R1][R5] 삭제 요청 대행의 건당 과금은 케이스 연동 변동비를 회수하기 위한 입력값으로 설계하며, 외부 플랫폼의 삭제 요청 절차 수행과 결과 추적이 케이스 원가의 직접 동인으로 반영된다.[R4][R1]

단위경제 산출 절차는 결제 로그에서 매출을, 제휴 정산 리포트에서 제휴 매출을, 케이스 로그에서 케이스 처리량을, 비용 리포트와 외부 청구서에서 변동비와 고정비를 취합하고 동일 기간으로 정렬하는 방식으로 고정한다.[R1] 총마진은 매출총이익을 매출로 나눈 값으로 정의하며, 매출총이익은 매출에서 변동비 합계를 차감해 산출한다.[R1] 손익분기는 고정비를 ARPU와 총마진율로 나누는 산식으로 정의하여, 고정비와 단위경제가 실측으로 전환되는 즉시 손익분기 유료가입자 수를 도출하도록 설계한다.[R1]

지표 정의(계산식) 상태 현재값 또는 측정상태 벤치마크 또는 근거값 근거 [R#]
ARPU ARPU = 개인 구독 월 매출 합계 / 유료 개인 구독자 수 확인 불가 결제 로그와 구독 상태 로그 연계로 산출, 추가 자료 미제공으로 확인 불가 요금제 월요금과 믹스 가중합으로 산출 [R1] [R1]
총마진율 총마진율 = 매출총이익 / 매출 확인 불가 비용 리포트에서 변동비 취합 후 결제 로그 매출과 기간 정렬, 추가 자료 미제공으로 확인 불가 변동비는 사용자 수 연동과 케이스 수 연동으로 구분 [R1] [R1]
CAC CAC = 유입·획득·온보딩 관련 비용 합계 / 신규 유료 가입자 수 확인 불가 마케팅 지출 리포트와 결제 로그 결합, 추가 자료 미제공으로 확인 불가 비용 범주와 귀속 규칙은 회계 리포트로 확정 [R1] [R1]
LTV LTV = ARPU × 총마진율 × 평균 유료 유지기간 확인 불가 구독 상태 로그로 유지기간 산출, 추가 자료 미제공으로 확인 불가 유지기간 산출은 결제일과 해지일 기준 [R1] [R1]
Payback Payback = CAC ÷ ARPU ÷ 총마진율 확인 불가 CAC와 ARPU, 총마진율이 실측으로 전환되면 자동 산출, 추가 자료 미제공으로 확인 불가 산출 단위는 월 기준으로 고정 [R1] [R1]
손익분기 유료가입자 수 손익분기 유료가입자 수 = 고정비 ÷ ARPU ÷ 총마진율 확인 불가 고정비 리포트와 ARPU, 총마진율 결합, 추가 자료 미제공으로 확인 불가 고정비는 개발·운영 공통 비용으로 정의 [R1] [R1]
ARPA ARPA = 기업·제휴 정산 매출 / 유효 계약 수 확인 불가 제휴 정산 리포트 및 계약서 기반 산출, 추가 자료 미제공으로 확인 불가 계약 단위는 연간 계약 및 정산 주기 기준 [R1] [R1]
무료 체험 유료 전환율 전환율 = 무료 체험 가입자 중 유료 전환자 수 / 무료 체험 가입자 수 2024.10 가정 [R1] 15-20% [R1] 내부 가정으로만 사용, As-of 실측은 결제 로그 미제공으로 확인 불가 [R1] [R1]

투자자 관점의 의사결정 포인트는 단위경제 표의 확인 불가 항목을 실측으로 전환할 수 있도록 결제 로그, 케이스 로그, 비용 리포트, 외부 청구서의 결합 권한과 산출 규칙이 통제되는지 여부다.[R1] 또한 VIP 구성의 공급자는 보험과 사고 대응 지원을 제공하는 형태의 공식 안내 범위 내에서 설계하되, 구체 보장 한도와 면책 조건은 약관 원문이 추가로 제공되기 전까지 확정 표현을 배제한다.[R5][R1] 케이스 단위 운영과 과금·원가 단위의 일치가 유지될 때, 월 구독료는 사용자 수 연동 변동비를, 건당 과금은 케이스 연동 변동비를 각각 상쇄하는 구조로 손익 모델 입력값이 고정된다.[R1]

3-1-5. 성과 지표 및 관리 체계(지표 정의·데이터 원천·보고 주기, 성과 지표 표 포함)
