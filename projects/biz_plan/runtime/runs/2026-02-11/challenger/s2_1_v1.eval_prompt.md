2-1. 서비스 현황 (준비 정도)

AI기반 마이디지털정보 오남용 알림 서비스는 사용자가 인지하지 못한 개인정보와 사진·동영상의 노출·유통을 탐지해 알림과 조치 착수로 연결하는 구독형 보호 서비스로 정의한다. 문제는 노출이 발생하는 채널이 분산되어 있고 재게시·재가공이 빠르게 일어나 사용자가 사후에 사실을 확인하는 방식으로는 대응의 일관성을 유지하기 어렵다는 점이다. 이에 따라 본 서비스는 탐지 범위를 멀티채널로 확장하는 대신, 탐지 신호의 신뢰도 관리와 사용자 조치 흐름의 표준화를 우선 과제로 두고 준비 수준을 단계별 산출물로 정리해 왔다.

가. 개발 이력/추진 경과
서비스 기획 착수 단계에서는 개인정보 및 이미지·영상 오남용이 발생하는 대표 경로를 유형화하고, 사용자 관점에서 노출 인지 지연과 조치 착수 지연이 어디에서 발생하는지 과업 흐름으로 정리했다. 이어 시장조사 및 경쟁분석을 통해 기존 대안이 수동 검색과 사후 신고에 편중되어 있으며, 멀티채널 탐지와 조치 지원을 통합적으로 제공하는 경험이 제한적이라는 가설을 도출하고 요구사항으로 반영했다. 다음 단계로 AI 모델 PoC를 통해 탐지 신호 생성의 기술적 실현성을 확인하고, 탐지 결과를 알림과 조치 착수로 연결하는 최소 사용자 흐름을 정의했다. 현재는 MVP 개발 착수 단계로, 탐지 파이프라인과 사용자 UI를 함께 설계하여 알림의 해석 가능성과 조치 안내의 일관성을 문서화하고, 운영 기준과 예외 처리 원칙을 제품 요건으로 고정하는 작업을 진행 중이다.

나. 현재 개발 단계(TRL 기준) 및 기술 검증
Technology Readiness Level은 기술의 성숙도를 단계적으로 정의하고, 실험실에서의 검증을 넘어 실제 적용 맥락에 가까운 환경에서의 통합 검증으로 이동하는 체계를 제공한다.[R1] 신청 시점의 본 서비스는 관련 환경에서 통합 동작을 검증하는 단계로 정의하며, 이는 핵심 구성요소가 단독 기능을 넘어 데이터 흐름과 운영 제약을 포함한 조건에서 함께 작동하는지 확인하는 수준을 의미한다. 얼굴 탐지 AI는 사용자 제공 샘플과 정책 기반 허용 범위를 전제로 모델 입력·출력과 경보 규칙의 결합을 검증했고, 개인정보 패턴 매칭은 텍스트 기반 식별정보와 계정·연락처 유형의 규칙을 정리해 탐지 후보를 생성하는 과정을 검증했다. 다크웹 수집·크롤링은 접근 경로의 불안정성과 중복 데이터가 발생하는 조건을 고려해 수집 파이프라인의 재시도와 저장 전 정규화, 증거 보존 형태의 산출물 생성을 중심으로 검증했으며, 알림 속도와 오탐 억제는 MVP 고도화 항목으로 분리해 개선 우선순위를 두고 있다.

구분 TRL 기준 정의 현재 상태 검증 산출물 다음 게이트
개념 및 요구사항 정립 문제·사용 시나리오와 요구사항을 정의 오남용 유형과 사용자 조치 흐름을 단계화 요구사항 명세, 위협·오남용 시나리오 목록 탐지 신호의 입력 규격 고정
실험실 수준 구현 및 기능 검증 요소기술 구현과 단위 기능 검증 얼굴 탐지, 패턴 매칭, 수집 모듈을 분리 검증 모듈별 테스트 리포트, 라벨링 가이드 통합 파이프라인 연결
관련 환경에서 통합 검증 적용 맥락에 가까운 조건에서 통합 동작 검증 탐지-알림-조치 착수 흐름을 MVP로 통합 검증 알림 정책 초안, 증거 보존 포맷, 운영 예외 규칙 실사용 조건에서 피드백 반영

다. 파일럿/테스트 결과 및 목표시장 반응
내부 테스터 기반 테스트에서는 사용자가 실제로 원하는 가치는 개별 기능의 성능 자체보다, 탐지된 신호가 무엇을 의미하는지 이해하고 다음 행동으로 즉시 이어질 수 있게 하는 연결성에 있다는 점이 반복적으로 확인되었다. 특히 알림이 많아질수록 사용자는 우선순위와 근거의 설명을 요구했으며, 단순 경고보다 조치 방법을 단계로 제시하고 증거를 정리해 주는 흐름을 선호했다. 또한 모니터링 범위에 대한 사용자 통제, 민감정보 최소 노출, 오탐 발생 시 되돌림 절차와 같은 신뢰 장치가 제품 완성도의 판단 기준으로 제시되었다. 이에 따라 개선 요구사항은 알림 속도와 정밀도 수치 경쟁이 아니라, 알림 해석 가능성, UI의 조치 착수 동선, 채널별 예외 처리, 사용자 설정과 로그 제공의 순서로 우선순위를 정해 반영하는 방향으로 정리했다.

라. 현재까지 주요 성과 및 다음 단계
현재까지의 성과는 주요 AI 모델 PoC 완료와 MVP 개발 착수에 더해, 탐지 결과를 조치 착수로 연결하는 운영 규칙과 증거 보존 형태를 문서화하여 제품화 과정에서 필요한 검증 항목을 사전에 정리했다는 점에 있다. 개인정보보호법 준수 관점에서는 개인정보 최소 수집과 목적 제한을 전제로, 탐지에 필요한 데이터와 사용자 식별정보를 분리 보관하고 접근통제 및 접속기록 관리를 포함하는 아키텍처 원칙을 설계에 반영했다.[R3] 대외적으로는 한국인터넷진흥원 지원사업 체계에 맞춰 기술 검증 산출물과 보호조치 항목을 정렬해 외부 검토가 가능한 문서 수준으로 정리했으며, 이를 통해 신청 이후의 실증·고도화 단계에서 점검해야 할 체크포인트를 운영 관점으로 구체화했다.[R2] 다음 단계는 MVP 고도화로 귀결되며, 알림 속도 개선과 UI 직관성 보완, 오탐 대응 절차의 명확화, 채널별 탐지 정책의 세분화를 통해 실사용 조건에서의 반복 검증을 확장하고, 조치 착수 기능을 삭제 요청 대행과 안내 템플릿으로 체계화하는 방향으로 진행한다.

[내용 평가 (80점)]
1. 완성도 (20점): 양식의 모든 필수 항목을 빠짐없이 포함했는가
2. 구체성 (20점): 수치, 데이터, 출처가 구체적인가 (모호한 표현 없는가)
3. 논리성 (20점): 논리적 흐름이 자연스럽고 설득력이 있는가
4. 차별성 (20점): 경쟁 서비스 대비 명확한 우위가 있는가

[형식 평가 (20점)]

6. 문서형식 (10점): 표가 마크다운 형식으로 올바르게 작성되었는가
