1-3. 목표시장(고객) 현황 분석

AI기반 마이디지털정보 오남용 알림 서비스는 사용자가 인지하지 못한 개인정보 및 사진·동영상의 노출·유통을 멀티채널에서 탐지하고, 알림과 조치(삭제 요청 대행 포함)로 연결하는 소비자 구독형 보호 서비스로 정의한다.[R7] 목표시장 분석의 기준은 “노출 인지 지연”이 발생하는 소비자 상황과, 이를 현재 대안(수동 검색·단일 범위 확인·비소비자용 도구)이 완전히 해소하지 못하는 구조적 제약을 팩트 기반으로 확인하는 데 둔다.[R3][R4][R6] 본 섹션의 수치는 상위 시장(TAM)→국내 인접시장(SAM)→초기 타겟(SOM) 순으로 의사결정에 활용하되, 소비자 구독 지출과의 정합성 한계를 별도로 명시하고 Consumer-SAM 및 Bottom-up 검증 구조로 보완한다.[R2][R7]

가. 시장 정의 및 범위(TAM/SAM/SOM)와 수치 적용 한계

구분 정의/범위 2024년 시장규모 본 서비스와의 관계
TAM 글로벌 디지털 아이덴티티 솔루션 시장 약 500억 달러[R1] 개인 식별·인증·리스크 관리 지출을 포함한 상위 지출 풀로서 참조 상한[R1]
SAM 국내 개인정보 보호 서비스 시장 약 8,000억원[R2] 국내 보호 지출의 인접시장 상한으로 활용하되, 소비자 구독과의 범위 차이를 별도 분리[R2]
SOM 초기 타겟(10-30대 SNS 헤비유저) 소비자 구독 시장 약 800억원[R7] 초기 실행 범위의 목표시장으로 사용하며, 퍼널 변수 실측으로 연동 갱신[R7]

TAM은 디지털 신원·보호 관련 상위 지출 풀로서 시장의 방향성과 벤더 생태계를 관찰하는 상한 지표로 활용하되, 본 서비스의 “소비자 구독형 노출 탐지·조치 대행”과 완전 동일 시장으로 단정하지 않는다.[R1][R7] SAM은 국내 개인정보 보호 서비스 시장 규모로 제시되지만, 지출 주체(소비자/기업/기관)와 제공 형태(컨설팅·솔루션·운영 등)가 혼재될 수 있으므로 소비자 구독 인접 지출로 재해석하는 구간을 별도로 둔다(함의).[R2][R7] SOM은 초기 타겟(10-30대 SNS 헤비유저) 범위에서 실행과 측정이 가능한 목표시장으로 사용하며, 단일 침투율 가정이 아니라 채널·전환·유지·ARPU로 분해해 검증 가능한 가정으로 운용한다.[R7]
의사결정 포인트: 10-30대 SNS 헤비유저가 노출 인지 지연을 겪는 상황에서, 수동 검색 중심 대안의 제약을 넘어 “알림 트리거+조치 대행”에 구독 지출이 발생하는지(전환·유지·ARPU)로 SOM의 타당성을 판단한다.[R3][R7]

나. 시장 규모(TAM/SAM/SOM) 및 Consumer-SAM 분리와 교차 검증 구조(Top-down/Bottom-up)

항목 근거(분류/범위) 수치 또는 변수식 의사결정 사용 방식
SAM(공식) 국내 개인정보 보호 서비스 시장(산업 조사 범주 기반)[R2] 8,000억원[R2] 국내 인접 지출 풀의 상한으로 사용[R2]
Consumer-SAM(인접) SAM 내 지출 주체 혼재를 고려해 소비자 구독 지출만 분리(함의)[R2] Consumer-SAM = SAM × α_consumer[R7] α_consumer 실측 전에는 범위 감도분석의 기준으로 사용[R7]

SAM은 국내 개인정보 보호 서비스 시장의 상한값으로 유용하지만, 소비자 구독 지출만을 의미한다고 단정하지 않으며 Consumer-SAM을 별도 변수로 둔다.[R2][R7] Consumer-SAM의 α_consumer는 (1) 소비자 구독형 서비스에 실제 지출이 발생한 거래 데이터(결제/구독 로그)와 (2) 세그먼트별 지불의사 및 대체재 사용 행태 조사로 실측해 업데이트하는 구조로 설계한다.[R7] Top-down은 “SAM→Consumer-SAM→초기 타겟 도달”로, Bottom-up은 “A(타겟 모수)×B(문제경험률)×C(유료전환율)×E(유지율)×D(ARPU)”로 서로 교차 검증하며, 두 관점의 결과가 괴리될 때는 α_consumer 및 퍼널 변수의 측정 우선순위를 조정한다(함의).[R7]
의사결정 포인트: 2차(유명인, 인플루언서, 크리에이터)와 3차(개인정보 민감 직군)가 지출을 견인할 수 있는지(ARPU·유지), 1차(10-30대 SNS 헤비유저)가 규모를 견인할 수 있는지(도달·전환)로 Consumer-SAM의 α_consumer를 실측해 상한(SAM) 대비 현실적 실행 범위를 정한다.[R7]

다. SOM 산정 드라이버(퍼널 분해: A/B/C/D/E)와 시나리오(보수/기준/공격)

변수 정의 보수/기준/공격(범위 또는 조건) 근거/출처 측정/수집 방식
A(타겟 모수) 초기 타겟 사용자 모수(10-30대 SNS 헤비유저)[R7] 보수: A=A_low[R7] / 기준: A=A_mid[R7] / 공격: A=A_high[R7] 내부자료[R7] 광고 플랫폼 도달·앱스토어 유입·온보딩 완료 수 기반으로 산출[R7]
B(문제경험률) 기간 내 “본인 비인지 노출” 경험 또는 의심 이벤트 경험 비율[R7] 보수: B=B_low[R7] / 기준: B=B_mid[R7] / 공격: B=B_high[R7] 내부자료[R7] 온보딩 설문+탐지 이벤트 로그(알림 발생)로 교차 확인[R7]
C(유료전환율) 무료 체험/알림 경험 이후 유료 전환 비율[R7] 보수: C=C_low[R7] / 기준: C=C_mid[R7] / 공격: C=C_high[R7] 내부자료[R7] 결제 전환 퍼널(방문→가입→설정→알림→결제) 로그로 산출[R7]
E(유지율) 구독 유지 비율(월 유지/분기 유지 등)로 정의[R7] 보수: E=E_low[R7] / 기준: E=E_mid[R7] / 공격: E=E_high[R7] 내부자료[R7] 해지 코호트 분석+알림 품질 지표(오탐/재탐)와 연동[R7]
D(ARPU) 유료 사용자 평균 과금(패키지별 과금 단가×구독 기간)로 정의[R7] 보수: D=D_low[R7] / 기준: D=D_mid[R7] / 공격: D=D_high[R7] 내부자료[R7] 패키지별 결제 단가·업셀·할인 로그로 산출[R7]
SOM SOM = A×B×C×E×D 최종 SOM=800억원[R7] 내부자료[R7] 변수 실측 후 업데이트[R7]

SOM은 단일 침투율을 고정하는 방식이 아니라, 퍼널 변수(A/B/C/E/D)의 조건 조합으로만 산정하고 실측 데이터로 갱신한다.[R7] 시나리오의 “보수/기준/공격”은 숫자 제시가 아니라 변수 조건의 조합으로 정의해, 광고·온보딩·알림 품질·해지 사유에 의해 변수가 움직일 때 어떤 항이 민감하게 반응하는지 확인한다.[R7] 특히 C(유료전환율)과 E(유지율)는 대체재 대비 “자동 탐지 알림”과 “조치 대행”이 제공하는 체감 가치에 의해 결정되므로, 경쟁/대체재의 구조적 제약을 근거로 검증 항목을 우선 배치한다(함의).[R3][R6][R7]
의사결정 포인트: 1차는 채널 도달(A)과 전환(C), 2차·3차는 유지(E)와 ARPU(D)를 견인하는 구조로 가설을 분리하고, 수동 검색·비소비자용 도구의 제약을 넘어서는 구독 트리거가 실제로 나타나는지(알림 후 조치 착수율·해지 사유)로 판단한다.[R3][R6][R7]

유명인, 인플루언서, 크리에이터[R7] 콘텐츠 무단 전재·편집본 확산으로 브랜드/수익 손실, 반복 모니터링 부담 증가[R7] 사전 경고(내가 올린 사진 기반 위험 신호)[R7]; 얼굴/유사도 기반 자동 탐지 알림[R7]; 삭제 요청 대행 및 증빙 제공[R7] MCN/에이전시 제휴 유입 설계[R7]; 크리에이터 커뮤니티 제휴 유입 설계[R7] 삭제 요청 처리 지연 체감[R7]; 증빙 부족으로 분쟁 대응 부담 지속[R7]; 과금 대비 탐지 범위 체감 저하[R7] 크리에이터[R7]
개인정보 민감 직군(변호사, 의사, 공무원 등)[R7] 자격·직함·연락처 결합 노출로 사칭·피싱 위험 증가, 개인 단위에서 상시 확인 부담 발생[R7] 개인정보+사진/영상 통합 모니터링[R7]; 얼굴/유사도 기반 자동 탐지 알림[R7]; 삭제 요청 대행 및 증빙 제공[R7] 전문직 단체/커뮤니티 제휴 유입 설계[R7]; 검색 기반 유입 설계[R7] 조치 대행 범위 제약 체감[R7]; 알림의 설명력 부족(무엇을 왜 탐지했는지)[R7]; 내부 규정과 상충되는 사용 정책[R7] 프로[R7]

의사결정 포인트: 10-30대 SNS 헤비유저는 반복 수동 검색의 제약을 넘어 “알림 기반”에 비용을 지불하는지(C), 유명인·전문직은 조치 대행의 운영 품질이 유지(E)와 ARPU(D)에 반영되는지로 패키지 우선순위를 결정한다.[R3][R7]

바. 차별화 포인트(기술/데이터/운영)와 운영 리스크(삭제 요청 대행/SLA/스코프 단계화)
본 서비스의 차별화는 멀티채널 통합 스캔(SNS+웹+다크웹)과 통합 모니터링, 얼굴/유사도 기반 자동 탐지와 알림 트리거, 유출 콘텐츠 삭제 요청 대행 및 운영 SLA/증빙 제공의 3축으로 고정한다.[R7] 기술 관점에서는 멀티채널 크롤링·수집 파이프라인, 얼굴/유사도 매칭, 알림 트리거를 구성요소로 두고 Precision/Recall, 탐지 리드타임, 오탐률 같은 성능 지표 이름으로 품질을 관리한다.[R7] 데이터 관점에서는 개인정보(식별·연락·자격 단서)와 사진/영상 신호를 동일 사용자 단위로 묶어 중복·동명이인·유사 이미지 충돌을 관리하는 정책을 설계하고, 채널별(웹/SNS/다크웹) 데이터 품질 차이를 태그로 기록해 알림 설명력을 제공한다.[R7] 운영 관점에서는 삭제 요청 대행 흐름을 접수→증빙→요청→추적→완료보고로 표준화하고, 반려 시 재요청·에스컬레이션·대체 조치 안내를 케이스 유형으로 분기한다.[R7] 운영 SLA는 처리 리드타임, 완료율, P50/P90 리드타임, 반려율, 재발견률을 지표로 정의하고 수치 목표는 내부 기준 확정 이후에만 반영한다.[R7] 스코프 단계화는 1단계(MVP)에서 탐지/알림 중심으로 오탐률·탐지 리드타임·알림 후 조치 착수율을 측정하고, 2단계에서 채널 확대 및 삭제 요청 대행 고도화를 통해 운영 KPI(처리 리드타임·완료율·P50/P90·재발견률)를 안정화하는 순서로 설계한다.[R7]
의사결정 포인트: 크리에이터·전문직이 조치 대행과 증빙을 비용 지불의 직접 사유로 삼는지(유지 E, ARPU D), 10-30대 SNS 헤비유저가 알림 품질(오탐/설명력)을 전환의 직접 사유로 삼는지(C)로 1단계(MVP) 우선순위를 확정한다.[R7]

사. 시장 상황·경쟁 강도·향후 관찰 포인트(근거 기반: 수치/범주/제약/함의)
시장 상황은 “소비자 접근 가능 도구의 목적 분절”과 “비소비자용 고성능 얼굴검색 도구의 존재”로 요약되며, 이는 소비자 구독 시장에서 통합형 서비스가 제공해야 할 기준선을 ‘멀티범위+상시 알림+조치’로 끌어올린다(함의).[R3][R6][R7] SAM 8,000억원은 국내 개인정보 보호 서비스 지출의 인접 상한으로 사용하되, Consumer-SAM(소비자 구독 인접)을 별도 변수로 두어 지출 주체 혼재를 제거한 뒤에만 구독형 매출 예산과 마케팅 효율 지표를 연결한다.[R2][R7] 향후 관찰 포인트는 (1) 이미지 수동 검색 대체에서 자동 알림으로의 전환이 C(유료전환율)를 움직이는지, (2) 삭제 요청 대행의 처리 품질이 E(유지율)를 움직이는지, (3) 세그먼트별 패키지 차등이 D(ARPU)를 움직이는지로 정리하며, 이는 SOM을 퍼널 실측으로 재계산하는 규칙에 직접 연결된다.[R7]
의사결정 포인트: 10-30대 SNS 헤비유저는 “상시 알림”에서, 크리에이터·전문직은 “조치 대행+증빙”에서 비용 지불이 발생하는지로 목표시장 우선순위를 정하고, 대체재의 제약(수동/단일 범위/비소비자 제공)을 넘는 지표 변화(전환·유지·조치 성과)를 기준으로 SOM 800억원의 실행성을 판단한다.[R3][R6][R7]
